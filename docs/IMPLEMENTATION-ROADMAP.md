## 2. 交付策略

### 2.1 我们不再把“模板广场 + 表单”当作 MVP 终点

从这版开始，MVP 的判断标准应该是：

1. 商场端能做模板
2. 模板能审核
3. 模板能授权
4. 商铺端能通过对话用模板
5. 结果能导出用于广告机

也就是说：

**只有商铺侧的对话编辑不够，必须补齐商场后台这半边。**

### 2.2 分阶段原则

建议按 P0 / P1 / P2 推进：

1. P0：形成真正业务可跑通的 MVP
2. P1：补强体验和自动化能力
3. P2：做增强型智能能力

---

## 3. P0：真正的最小可用 MVP

### 3.1 商场后台必须补齐

P0 必须完成：

1. 分类管理页
2. 模板库页
3. 模板编辑器页
4. 审核列表页
5. 模板授权页

### 3.2 商场端 AI 制模板能力

P0 范围内必须有：

1. 基于场景和对话生成背景
2. 基于对话在背景上生成文字区、价格区、二维码区、装饰区
3. 模板保存为标准 JSON
4. 模板预览

P0 暂不要求：

1. 参考图逆向工程的完整 Vision 复刻

### 3.3 审核与授权

P0 必须完成：

1. 模板状态流：`Draft -> Pending -> Approved / Rejected`
2. 审核意见记录
3. 模板授权给指定商铺
4. 商铺端只看到已授权且已审核模板

### 3.4 商铺端主链路

P0 必须保证：

1. 商铺登录后按权限看到模板
2. 进入对话编辑器
3. 替换店名、价格、文案、图片、二维码
4. 自动处理长短文字
5. 导出 H5 / 图片

### 3.5 P0 验收标准

满足以下五条才算 P0 完成：

1. 商场运营能创建一套模板并提交审核
2. 审核员能通过或驳回模板
3. 模板通过后，指定商铺可以看到它
4. 商铺能通过对话完成内容替换
5. 商铺能导出广告机可用内容

## 7. 推荐开发顺序

### 第一步

先补商场后台最小闭环：

1. 分类页
2. 模板库
3. 模板编辑器
4. 审核页

### 第二步

把模板从“种子模板 + 手工 JSON”推进到：

1. 可对话创建背景
2. 可对话添加结构
3. 可保存正式模板 JSON

### 第三步

补权限与授权：

1. 角色体系
2. 模板授权
3. 商铺过滤可见模板

### 第四步

完善商铺侧：

1. 版本持久化
2. 素材持久化
3. PNG 导出
4. 小程序集成参数

### 第五步

最后补高级智能：

1. 标签推荐
2. 文案生成
3. 参考图逆向工程

---

## 8. 近期建议

如果现在立刻继续开发，建议优先做这三个任务：

1. 商场后台模板编辑器 MVP
2. 模板审核 + 授权链路
3. 商铺端权限过滤后的模板大厅

原因很简单：

如果没有这三项，当前系统虽然已经有“对话编辑器”，但仍然没有真正进入“商场统一制模板、商铺按权限用模板”的业务闭环。
